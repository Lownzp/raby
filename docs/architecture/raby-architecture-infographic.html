Raby v0.1 Architecture / Local-first Flutter

兔兔档案应用架构总览

Presentation 只处理界面，Application 组织状态和命令，Domain 表达业务，Data 隔离 Drift/SQLite 与媒体文件。

分层职责

app路由、主题、Provider 装配、启动壳
features页面、表单、Controller、局部异步状态
domain实体、枚举、Repository 接口、业务异常
dataDrift DAO、Repository 实现、mapper、媒体存储
Flutter Riverpod go_router Drift SQLite

事务写入

数据模型核心

所有表都有 updated_at / deleted_at 语义，查询默认过滤软删除。媒体文件不进库，只保存相对路径。

依赖规则

推荐

  • UI → Application
  • Application → Domain
  • App Providers → Data 实现
  • Data → Domain mapper

禁止

  • Domain import Flutter/Drift
  • Presentation import DAO/Table
  • Data import features
  • Drift row 穿透 UI
