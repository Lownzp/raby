Boundary Rules

依赖边界是这套架构的价值核心

允许的方向

● app 装配 router、theme、repository providers。
● features/presentation 只依赖 application、shared、router。
● features/application 调用 domain repository 接口。
● data 实现 repository，负责 Drift row 与 Domain 转换。
● shared 只承载跨页面复用 UI 和 theme token。

禁止的穿透

● domain 不能 import Flutter、Riverpod、Drift 或 data。
● presentation 不能直接 import DAO、Table、Companion。
● Drift row 不能穿透到页面、表单或 Domain 接口。
● Repository 不处理导航，保存成功由页面响应状态。
● data 不依赖 features，避免功能层倒灌。
Data Core

SQLite 里保存事实，私有目录保存媒体

rabbits

默认兔、头像相对路径、建档资料；软删除会级联标记相关日记、标签、体重。

diaries

按 rabbit_id 与 recorded_at 组成时间线核心索引。

diary_media

只存相对路径和排序，文件位于 App 私有目录。

tags / diary_tags

系统标签 seed，自定义标签按兔归属，多对多连接日记。

weight_records

按兔和日期输出列表及 fl_chart 图表数据。

schemaVersion 1

建库创建索引，beforeOpen 打开外键约束。
