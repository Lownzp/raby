## 1. 建档/首次使用页

页面目的：新用户第一次进入 App，创建宠物档案。

结构：

- 状态栏。
- 居中标题：`创建宠物档案`。
- 副标题：`让我们更好地了解你的兔兔吧～`。
- 大兔子主视觉，占据上半屏视觉重心。
- 档案表单卡片。
- 说明条。
- 主按钮：`完成建档`。
- 次要入口：`稍后再说`。

组件：

- 大图主视觉。
- 头像上传。
- 文本输入：名字。
- 单选/选择输入：品种、生日/年龄。
- 胶囊选择：性别。
- 数值输入：初始体重。
- 说明条。
- 主按钮。

状态：

- 空表单。
- 已选择头像。
- 缺少必填项。
- 保存中。
- 保存失败。

还原重点：

- 该页无底部 Tab。
- 主视觉兔子图片要足够大，但不能压迫表单。
- 表单信息密度清晰，字段高度一致。
- 贴纸只保留胡萝卜或小兔之一，不额外堆装饰。

## 2. 首页

页面目的：让用户快速看到兔子当前状态，并进入最常用记录动作。

结构：

- 品牌顶部栏：Raby 标识、搜索、添加、设置。
- 主视觉宠物卡：兔子大图、宠物名称、简介、体重摘要和趋势入口。
- 快捷操作卡：`记体重`、`写日记`。
- 最近动态列表。
- 底部 Tab。

组件：

- 品牌 Logo。
- 圆形图标按钮。
- 主视觉宠物卡。
- 迷你折线图。
- 快捷操作卡。
- 动态列表项。
- 状态标签。
- 底部 Tab。

状态：

- 有数据。
- 无动态。
- 无体重记录。
- 图片加载失败。
- 长文本动态。

还原重点：

- 大兔子照片是视觉中心。
- 快捷操作卡必须在小屏保持横向可读。
- 最近动态使用真实照片缩略图，不使用插画替代照片。

## 3. 记体重页

页面目的：快速记录某一天的体重。

结构：

- 顶部导航：返回、标题、日历。
- 宠物信息卡：头像、名字、上次记录。
- 大数值输入卡：当前体重、加减按钮、刻度尺。
- 记录日期行。
- 变化摘要卡。
- 备注输入卡。
- 主按钮：`保存记录`。
- 底部 Tab。

组件：

- 顶部导航。
- 宠物信息卡。
- 数值输入器。
- 刻度尺。
- 日期选择行。
- 变化摘要卡。
- 备注输入框。
- 主按钮。

状态：

- 初始值。
- 数值变化。
- 无上次记录。
- 备注为空。
- 保存中。
- 保存失败。

还原重点：

- `1820g` 的数字要作为主视觉，不能拥挤或换行。
- 加减按钮和刻度尺要看起来可操作。
- 装饰贴纸只放在体重卡角落。

## 4. 写日记页

页面目的：记录当天日常、饮食、心情、健康和照片。

结构：

- 顶部导航：返回、标题、更多。
- 宠物日期卡。
- 类型切换：日常、饮食、心情、健康。
- 文本输入区。
- 照片上传区。
- 今日状态标签组。
- 体重记录提示条。
- 主按钮：`保存日记`。
- 次要入口：`存为草稿`。
- 底部 Tab。

组件：

- 顶部导航。
- 宠物日期卡。
- 分段标签。
- 大文本输入框。
- 图片上传格。
- 状态胶囊。
- 关联动作提示条。
- 主按钮。

状态：

- 空文本。
- 长文本。
- 已上传 1-3 张图片。
- 图片上传失败。
- 草稿。
- 未记录体重。

还原重点：

- 表单页要比首页更安静。
- 标签选中态清楚，但不要过度装饰。
- 图片上传格保持固定尺寸。

## 5. 我的页

页面目的：展示主人、宠物档案、统计和设置入口。

结构：

- 顶部标题：`我的`，右侧设置按钮。
- 主人信息区。
- 宠物资料主卡。
- 统计卡：日记数量、体重记录、连续记录天数。
- 第一组功能列表：宠物资料、提醒设置、健康档案、照片相册、数据导出。
- 第二组设置列表：通知设置、主题设置、关于 Raby。
- 底部 Tab，高亮 `我的`。

组件：

- 用户头像。
- 宠物资料卡。
- 统计三列卡。
- 设置列表组。
- 列表项图标。

状态：

- 无主人头像。
- 无宠物档案。
- 统计为 0。
- 设置项禁用。

还原重点：

- 这是功能入口页，装饰要比首页克制。
- 宠物资料卡可以保留贴纸，列表项不要贴纸化。
- 统计数字要清楚，避免被图标抢焦点。

## 6. 日记详情页

页面目的：查看一篇日记的完整内容、图片和当天状态。

结构：

- 顶部导航：返回、标题、更多。
- 详情卡头部：宠物头像、名字、日期、分类标签。
- 日记正文。
- 图片拼贴区：1 大图 + 2 小图。
- 今日状态摘要。
- 当天体重行。
- 底部操作：`编辑`、`再写一篇`。
- 底部 Tab。

组件：

- 详情头部卡。
- 正文段落。
- 图片拼贴网格。
- 状态摘要卡。
- 关联体重行。
- 双按钮操作区。

状态：

- 无图片。
- 1 张图片。
- 3 张以上图片。
- 长正文。
- 无体重记录。

还原重点：

- 图片区域要像内容展示，不像相册管理。
- 底部操作按钮要明显区分主次。
- 正文排版留足行高。

## 7. 体重趋势页

页面目的：查看体重趋势、周期统计和历史记录。

结构：

- 顶部导航：返回、标题、日历。
- 宠物当前体重摘要卡。
- 周期分段：7 天、30 天、90 天、全部。
- 趋势图表卡。
- 三列统计：最高、最低、平均。
- 变化分析卡。
- 记录明细列表。
- 底部 Tab，高亮 `体重`。

组件：

- 当前体重摘要卡。
- 分段控制。
- 折线图。
- 统计摘要。
- 分析提示卡。
- 体重记录列表项。

状态：

- 无记录。
- 记录少于 2 条。
- 体重上升。
- 体重下降。
- 体重稳定。
- 周期切换。

还原重点：

- 图表是主角，贴纸只能作为卡片辅助。
- 折线、数据点、末端标注必须清晰。
- 明细列表要可扫读，不能过度卡片化。

## 8. 照片相册页

页面目的：查看和管理兔子的照片记录。

结构：

- 顶部导航：返回、标题、添加。
- 宠物相册信息卡。
- 分类筛选：全部、日常、饮食、心情、健康。
- 三列照片网格。
- 底部 Tab，高亮 `我的` 或保持主 Tab。

组件：

- 相册信息卡。
- 分类胶囊。
- 照片网格。
- 日期角标。
- 收藏角标。

状态：

- 有照片。
- 无照片。
- 分类为空。
- 图片加载失败。
- 选择模式。

还原重点：

- 照片是绝对主角。
- 每张图片只保留必要日期标签，不给每张图加贴纸。
- 网格间距稳定，图片裁切统一。
