# Component System

## 组件分层

Raby 的组件分为四层：

1. App Shell：状态栏、顶部导航、底部 Tab、页面背景。
2. Content Cards：主视觉卡、宠物信息卡、表单卡、图表卡、设置列表卡。
3. Controls：按钮、胶囊标签、输入框、图标按钮、上传格、分段控制。
4. Decorative Assets：兔子照片、重点贴纸、少量碎装饰。

## App Shell

### 页面背景

- 使用暖白/奶油底色。
- 可以有极轻的纵向柔光，但不能形成强渐变海报感。
- 页面内容左右边距建议保持一致。
- 卡片之间需要有稳定垂直节奏。

### 顶部导航

适用：记体重、写日记、趋势、详情、相册、设置等二级页。

规则：

- 左侧为返回按钮。
- 中间为标题。
- 右侧为 0-1 个主要操作：日历、更多、添加、设置。
- 标题可有极少量装饰，但每页最多一次。
- 图标按钮使用圆形浅色底和统一线性图标。

### 底部 Tab

固定 3 个：

- 首页。
- 体重。
- 我的。

规则：

- 主流程页面显示底部 Tab。
- 建档/首次使用不显示底部 Tab。
- 二级功能页是否显示 Tab 以预览图为准：记体重、写日记、详情、相册显示；更深层设置页可不显示。
- 选中态使用橙黄色图标和浅黄色胶囊背景。

## Cards

### 主视觉宠物卡

适用：首页。

内容：

- 宠物名称。
- 年龄/品种简介。
- 大兔子照片。
- 当前体重摘要。
- 趋势入口。

规则：

- 兔子照片占卡片视觉面积 40%-55%。
- 文本和数据卡不能遮挡兔子面部。
- 主视觉卡允许 1-3 个重点贴纸。

### 宠物信息卡

适用：记体重、写日记、详情、相册、我的。

内容：

- 圆形头像。
- 宠物名。
- 次级信息：日期、上次记录、照片数量等。
- 可选右侧操作。

规则：

- 头像要有白色描边。
- 信息层级固定：名字最大，次级信息较小。
- 只有卡片角落允许放一个重点贴纸。

### 表单卡

适用：建档、记体重、写日记、宠物资料编辑。

规则：

- 字段高度统一。
- Label 和输入区域对齐。
- 输入框为浅底、浅描边、大圆角。
- 表单卡里不放碎装饰。
- 错误态用清楚的文案和轻量红色，不用夸张图形。

### 图表卡

适用：首页体重摘要、体重趋势。

规则：

- 图表优先级高于贴纸。
- 主线使用橙黄色。
- 网格线浅且低对比。
- 当前点、最高点、最低点要有明确可读标记。
- 图例只保留必要信息。

### 设置列表卡

适用：我的、设置、通知、主题。

规则：

- 每行包含图标、标题、右箭头或开关。
- 图标统一线性或轻量色块，不使用复杂贴纸。
- 行高稳定，分割线轻。
- 列表卡不加额外星星爱心。

## Controls

### 主按钮

适用：完成建档、保存记录、保存日记、再写一篇。

规则：

- 使用橙黄色填充。
- 文案居中，字重较高。
- 高度稳定。
- 按钮宽度根据上下文决定：页面主操作通常通栏，详情页可双按钮并排。

状态：

- Normal。
- Pressed。
- Disabled。
- Loading。

### 次按钮

适用：编辑、稍后再说、取消。

规则：

- 使用浅底或描边。
- 不能比主按钮更抢眼。
- 文案与主按钮保持同一语气。

### 胶囊标签

适用：分类、状态、周期、性别。

规则：

- 选中态使用浅黄色底和橙色文字。
- 未选中态使用白色或浅灰底。
- 图标可选，但不能每个标签都贴纸化。
- 横向空间不足时优先横向滚动，不能挤压成竖排。

### 输入框

适用：名字、备注、日记正文、体重。

规则：

- 文本输入区域要清楚可点击。
- 长文本输入框保留计数器。
- 体重数值输入使用大号数字，但必须在 360px 宽屏内不溢出。

### 图片上传格

适用：写日记、相册。

规则：

- 已上传图显示真实缩略图。
- 空位显示加号。
- 删除按钮只出现在已上传图角落。
- 尺寸固定，上传状态不能撑开布局。

## Decorative Assets

### 重点贴纸

可用对象：

- 胡萝卜。
- 小兔。
- 体重秤。
- 日记本。
- 相机。
- 花朵。
- 叶子。

规则：

- 每页最多 1-3 个。
- 优先放在卡片角、头像旁、空状态中心。
- 不放在每个列表项里。
- 不遮挡核心文本和照片主体。

### 碎装饰

对象：

- 小星星。
- 小爱心。
- 小闪光。

规则：

- 每页最多 0-2 处。
- 只用于标题附近或主卡片角落。
- 列表区、表单区、图片网格区禁用。

## 文案规则

- 动作用短动词：`保存记录`、`保存日记`、`完成建档`。
- 状态用短词：`稳定`、`平稳`、`开心`、`很好`、`活跃`。
- 空状态说明要给下一步动作，不做纯情绪表达。
- 同一动作在不同页面保持同名。
