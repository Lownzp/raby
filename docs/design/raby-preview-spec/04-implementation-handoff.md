# Implementation Handoff

## 开发顺序

### 阶段 1：设计系统基础

目标：先做统一组件和主题，不直接堆页面。

内容：

- 颜色、字体、圆角、阴影、间距 token。
- 页面背景。
- 顶部导航。
- 底部 Tab。
- 图标按钮。
- 主按钮和次按钮。
- 卡片基础样式。
- 胶囊标签。
- 输入框。

验收：

- 同一组件在不同页面尺寸、颜色、圆角一致。
- 360px 宽屏不出现文字竖排、挤压或溢出。

### 阶段 2：首页、记体重、写日记

目标：先完成最高频闭环。

内容：

- 首页展示当前宠物、体重摘要、快捷入口、最近动态。
- 记体重可输入、保存、更新趋势。
- 写日记可输入文字、上传/展示图片、选择状态、保存。

验收：

- 首页快捷入口在窄屏保持横向可读。
- 记体重数字 `1820g` 不换行。
- 写日记图片上传格尺寸稳定。
- 保存后首页和相关列表可更新。

### 阶段 3：体重趋势、日记详情、相册

目标：补齐查看和管理体验。

内容：

- 趋势页支持周期切换、图表、统计、明细。
- 日记详情支持图文展示、状态摘要、编辑和再写一篇。
- 相册页支持分类筛选、照片网格、日期角标和图片查看。

验收：

- 图表可读，末端值标注清楚。
- 多图详情布局不变形。
- 相册三列网格稳定，图片裁切统一。

### 阶段 4：我的、建档、设置衍生页

目标：补齐新用户入口和资料设置。

内容：

- 首次启动建档。
- 我的页资料、统计、功能入口。
- 宠物资料编辑、设置、通知、数据导出等衍生页。

验收：

- 无档案时进入建档流程。
- 建档完成后进入首页。
- 我的页统计和入口可点击。

## 设计还原标准

### 视觉 DNA

必须满足：

- 看到任一页面都能识别为 Raby。
- 真实兔子照片与奶油卡片构成主要视觉。
- 橙黄色作为主行动色，深棕作为文字主色。
- 贴纸少但有记忆点。

不能出现：

- 每个角落都有星星爱心。
- 列表项全部贴纸化。
- 页面像营销海报而不是工具 App。
- 照片被滤镜、遮罩或装饰压过主体。

### 组件一致性

必须满足：

- 同类卡片圆角一致。
- 同类按钮高度一致。
- 同类标签选中态一致。
- 顶部导航和底部 Tab 在页面间一致。
- 图标风格一致。

### 响应式

至少检查：

- 360 x 780。
- 390 x 844。
- 430 x 932。

重点检查：

- 首页快捷操作卡。
- 记体重大数字。
- 写日记标签和图片上传。
- 趋势页图表和明细。
- 相册三列网格。
- 我的页列表。

## 截图 QA

每轮 UI 开发后输出固定截图：

```text
home
weight-edit
diary-edit
weight-trend
diary-detail
album
profile
onboarding
settings
```

每张截图检查：

- 标题、按钮、正文无重叠。
- 文字不竖排，不截断关键语义。
- 贴纸不遮挡文本和图片主体。
- 卡片间距稳定。
- 底部 Tab 不遮挡内容。
- 图片加载后布局不跳动。

## 功能闭环

最低闭环：

1. 首次建档。
2. 首页查看宠物状态。
3. 记录体重。
4. 查看体重趋势。
5. 写日记。
6. 查看日记详情。
7. 查看相册。
8. 进入我的页管理资料和设置。

## 开发风险

| 风险 | 处理 |
| --- | --- |
| GPT 预览图尺寸夸张，代码实现后拥挤 | 以固定组件高度和响应式约束落地 |
| 贴纸越加越乱 | 每页限制贴纸数量，并分离功能图标 |
| 真实照片风格不统一 | 先建立照片素材库，统一裁切和色温 |
| 图表手写成本高 | 使用成熟图表库或封装单一折线图组件 |
| 小屏文字溢出 | 关键组件加截图测试和窄屏 widget test |

## 下一步动作

1. 把最终预览图文件归档到项目内可追踪目录。
2. 根据本规格抽取 Flutter theme 和通用组件。
3. 先实现组件展示页或局部页面，做第一轮截图对齐。
4. 再按阶段进入功能闭环开发。
