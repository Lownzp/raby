## 1. 产品定位

为**个人养兔玩家**打造的、**本地优先**的兔子档案 App,核心是「日记 + 体重 + 数据可视化」。

### 1.1 目标用户

- 家里养 1~5 只宠物兔的个人用户
- 默认单只(主推一只),允许添加多只
- 暂时不面向繁殖者、救助站、兽医

### 1.2 必须有 vs 锦上添花

| 优先级 | 功能 |
|---|---|
| **必须有(MVP)** | 生活记录(日记/相册);基础档案(静态信息);体重管理;数据分析(图表) |
| **锦上添花(v2+)** | 完整健康管理(疫苗/驱虫/就诊/用药);饮食作息/便便;家庭共享 |
| **明确不做** | 地点 / GPS;繁殖 / 配种 / 血统;社交(发现页、关注、互动) |

## 2. 平台与发布

- **平台**:Android 单端(暂不做 iOS / Web / 小程序)
- **最低版本**:暂定 Android 8(API 26),覆盖 99%+ 设备
- **分发方式**:暂定通过 APK 直接安装(后续可上 Google Play / 国内应用市场)

---

## 3. 数据存储策略

- **方案**:本地存储 + 手动导出(暂不做云同步,但要为未来上云预留架构)
- **数据库**:SQLite(通过 Drift)
- **媒体文件**:App 私有目录(`getApplicationDocumentsDirectory()`),避免被系统清理
- **导出**:打包成 ZIP(数据库 + 媒体 + 元数据),通过系统分享菜单让用户决定存哪
- **导入**:从 ZIP 还原,支持「合并 / 覆盖」两种模式

### 3.1 为「未来上云」做的关键铺垫

1. **所有数据用 UUID(v4),不用自增 ID** —— 多端合并不冲突
2. **所有表带 `updatedAt`、`createdAt`、`deletedAt`(软删除)** —— 同步用得上
3. **Domain 层不依赖 Flutter / Drift / Riverpod** —— Repository 接口在 Domain,实现在 Data,以后加 RemoteDataSource 不影响业务逻辑
4. **业务数据和媒体文件分离存储** —— 同步策略不同(JSON 用增量,媒体用 hash 去重)

## 4. 功能需求(已确认细节)

### 4.1 基础档案

支持记录字段:

**必填**:名字、性别、生日(或领养日)、品种、毛色

**可选**:头像照片、出生地/来源、绝育状态及日期、芯片号、初始体重、性格标签、喜好(食物/玩具)、离世日期(虹桥)

**不做**:家庭成员

### 4.2 体重管理

- **记录频率**:不固定;支持「自定义周期提醒」(可选,用户可关)
- **每次记录字段**:体重(g)+ 日期(必有)、备注、配照片、BCS 体况评分(1~5)
- **图表展示**:
  - 体重折线图,支持切换 周 / 月 / 年 / 全部
  - 关键事件竖线标记(绝育、生病等)
  - 异常预警(突变超过阈值,如 7 天内 ±10%)

### 4.3 生活记录(日记)

**单条日记构成**:
- 多张照片(单条上限 9 张,参考微信朋友圈)
- 短视频(单条上限 30 秒,自动压缩至 720p)
- 文字(不限长度,默认折叠 3 行)
- 标签(支持系统预置 + 用户自定义)

**不做**:地点 / GPS

**浏览方式**:
- 时间轴 feed 流(主)
- 「那年今日」回忆推送(特色功能,本地通知触达)

### 4.4 成长里程碑

**实现方式**:复用日记系统,通过「⭐里程碑」特殊标签实现,不做独立入口。

- 系统预置常见里程碑模板(第一次吃苹果、第一次剪指甲、第一次见兽医…)
- 在「兔生大事记」中只展示带⭐的日记

### 4.5 数据分析

- 📅 **记录热力图**:像 GitHub 贡献图,看每天有没有写日记
- 🎂 **兔生大事记**:自动汇总所有 ⭐里程碑 + 健康事件(绝育、生病)
- 🤖 **健康预警**:体重突变 / 长期未记录提醒

**不做**:标签云、活跃度趋势、照片占用统计(花哨但洞察弱)

## 5. 技术栈

| 用途 | 选型 | 理由 |
|---|---|---|
| 框架 | **Flutter** | 跨平台留扩展空间;媒体处理生态成熟;图表/列表/动画体验稳定 |
| 状态管理 | Riverpod 2.x | 类型安全、编译期检查、可测试 |
| 路由 | go_router | Google 官方,声明式 |
| 本地数据库 | Drift(SQLite ORM) | 类型安全、编译期 SQL 校验、迁移友好 |
| 文件存储 | path_provider + 自建目录管理 | App 私有目录,避免被清理 |
| 多图选择 | wechat_assets_picker | 微信同款 UI |
| 拍照/拍视频 | camera | 官方插件 |
| 视频压缩 | video_compress(+ 失败兜底) | 国产 ROM 偶尔失败,需要兜底原文件 |
| 图片浏览 | photo_view | 双指缩放、滑动切换 |
| 图表 | fl_chart | 折线/柱状/热力图,免费、活跃 |
| 本地通知 | flutter_local_notifications | 体重提醒、那年今日 |
| 数据导出 | archive + share_plus | ZIP 打包 → 系统分享 |
| 数据类 | freezed + json_serializable | 不可变、自动生成、可序列化 |
| 日期处理 | intl | 国际化、日期格式化 |
| 唯一 ID | uuid v4 | 为云同步预留 |
| 国际化 | flutter_localizations + intl | 中文为主,预留英文 |
| 日志 | logger | 分级日志 |

### 5.1 为什么选 Flutter

1. **媒体处理稳定**:相机、多图选择、视频压缩在 Android 上踩坑最少
2. **未来扩展性强**:今天写的代码以后加 iOS / 小程序成本低
3. **图表与列表性能好**:大量照片缩略图列表自绘渲染,不掉帧
4. **国产生态成熟**:闲鱼 / 字节 / 京东等大厂在生产环境用

## 6. 架构设计

### 6.1 分层(Clean Architecture 简化版)

```
Presentation Layer (Widgets + Riverpod + go_router)
        ↓
Domain Layer (Entities + UseCases + Repository 接口)   ← 纯 Dart,无 Flutter 依赖
        ↓
Data Layer (Repository 实现 + Drift + 文件存储)         ← 未来追加 RemoteDataSource
```

### 6.2 项目目录结构

```
lib/
├── main.dart
├── app/                       # 应用级配置
│   ├── app.dart               # MaterialApp 入口
│   ├── router.dart            # go_router 配置
│   └── theme.dart             # 主题
├── core/                      # 通用工具
│   ├── constants/
│   ├── utils/
│   ├── errors/                # Failure / Exception 定义
│   └── extensions/
├── data/                      # 数据层
│   ├── database/              # Drift 数据库
│   │   ├── app_database.dart
│   │   ├── tables/            # 表定义
│   │   └── daos/              # 数据访问对象
│   ├── storage/               # 文件存储抽象
│   │   └── media_storage.dart
│   └── repositories/          # Repository 实现
├── domain/                    # 领域层(纯 Dart)
│   ├── entities/              # freezed 实体
│   ├── repositories/          # 接口
│   └── usecases/              # 业务用例
├── features/                  # 业务模块(按特性切)
│   ├── rabbit_profile/        # 基础档案
│   ├── diary/                 # 日记
│   ├── weight/                # 体重
│   ├── analytics/             # 数据分析
│   └── settings/              # 设置(含导出导入)
├── shared/                    # 跨模块共享 UI
│   ├── widgets/
│   └── providers/             # 全局 Provider
└── l10n/                      # 国际化
```

### 6.3 关键架构决策

1. **Clean Architecture 简化版**,为未来上云铺路
2. **Riverpod + Drift + go_router** 黄金三件套
3. **按 feature 切分目录**,而不是按技术分层
4. **所有数据用 UUID 而非自增 ID** ——「未来云同步」的基础,不能后期改

### ✅ 已确认
- 个人养兔玩家;单只为主,允许多只
- Android 单端;Flutter

- 标签云 / 活跃度趋势图 / 照片占用统计(花哨弱洞察)
