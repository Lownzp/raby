## 1. 设计目标

Raby 是一个本地优先的兔子档案 App。界面目标不是社交、营销或宠物社区,而是让用户每天愿意轻松记录,并在需要时快速回看。

核心体验关键词:

- 温暖
- 安静
- 可信
- 易记录
- 易回看
- 不幼稚

设计上要避免两类偏差:

- 过度可爱:大量插画、贴纸、emoji、花哨装饰会降低长期记录工具的耐看度。
- 过度医疗:冷冰冰的医院蓝绿、复杂指标会让用户以为这是诊断工具。

## 2. 信息架构

### 2.1 v0.1 导航

v0.1 先使用 3 个底部导航入口:

| 入口 | 页面 | 目标 |
|---|---|---|
| 记录 | 日记时间轴 | 看最近生活记录,快速新增日记 |
| 体重 | 体重首页 | 看体重趋势,快速新增体重 |
| 我的 | 档案和设置 | 管理兔子档案和 App 设置 |

原因:

- v0.1 不做热力图、大事记和异常预警,不应放一个空的“分析”入口。
- v0.2 增加分析能力后,底部导航扩展为 4 项:记录、体重、分析、我的。
- 路由层可以提前保留分析路由,但 UI 不展示不可用入口。

### 2.2 v0.2+ 导航

| 入口 | 页面 | 目标 |
|---|---|---|
| 记录 | 日记时间轴 | 日常记录主入口 |
| 体重 | 体重首页 | 体重趋势和记录 |
| 分析 | 分析首页 | 热力图、大事记、预警 |
| 我的 | 档案和设置 | 兔子档案、备份、偏好 |

底部导航最多 4 项,不超过 5 项。

## 4. 视觉方向

### 4.1 风格

采用“温暖记录工具”风格,完整视觉规范见 [Raby 视觉风格规范](./2026-06-08-raby-visual-style-guide.md):

- 页面结构清爽,以内容和表单为主。
- 使用少量温暖色表达日记感,用冷静的绿色/青绿色提供可信感。
- 卡片和列表克制,不做过重阴影。
- 插画只用于空状态和首次建档,不成为主视觉负担。
- App 内图标统一使用 Raby 手绘线稿风格,保持轻手绘但可快速识别。
- 不使用 emoji 作为导航、按钮或状态图标。

### 4.2 色彩 Token

`ui-ux-pro-max` 对日记类产品推荐了 warm journal brown + ink violet。Raby 采用这个方向,但把主色调整为青绿色,避免界面变成单一棕橙/奶油色主题。

| Token | 颜色 | 用途 |
|---|---|---|
| `primary` | `#2F6F64` | 主按钮、选中态、关键进度 |
| `onPrimary` | `#FFFFFF` | 主色上的文字和图标 |
| `secondary` | `#92400E` | 日记感强调、里程碑、少量重点 |
| `onSecondary` | `#FFFFFF` | 次色上的文字和图标 |
| `accent` | `#6366F1` | 图表辅助、高亮信息 |
| `background` | `#FAFAF7` | 页面背景 |
| `surface` | `#FFFFFF` | 卡片、表单、底部栏 |
| `surfaceMuted` | `#EEF3F0` | 次级区域、筛选背景 |
| `border` | `#DDE7E2` | 分割线、输入框边框 |
| `textPrimary` | `#17211D` | 主文字 |
| `textSecondary` | `#64748B` | 次级文字 |
| `success` | `#059669` | 成功提示 |
| `warning` | `#B45309` | 体重关注提示 |
| `danger` | `#DC2626` | 删除、错误 |

规则:

- 页面背景不使用大面积渐变。
- 主色只用于主操作和选中态,不要满屏铺色。
- 警告和异常提示必须同时有文字说明,不能只靠颜色。
- 色彩对比需满足普通文字 4.5:1。

### 4.3 字体

v0.1 使用 Flutter/Android 系统字体,优先保证性能和平台一致性。

字号建议:

| 层级 | 字号 | 字重 | 用途 |
|---|---:|---:|---|
| Page Title | 24sp | 700 | 页面主标题 |
| Section Title | 18sp | 600 | 分区标题 |
| Card Title | 16sp | 600 | 卡片标题 |
| Body | 15-16sp | 400 | 正文 |
| Label | 13-14sp | 500 | 表单标签、Chip |
| Caption | 12-13sp | 400 | 辅助说明 |

规则:

- 正文不小于 15sp。
- 支持系统字体缩放,文本允许换行。
- 不使用负字距。
- 页面内不要使用 hero 级超大字。

## 5. 布局规范

### 5.1 间距

采用 4/8dp 间距系统:

| Token | 值 | 用途 |
|---|---:|---|
| `spaceXs` | 4dp | 图标和文字小间距 |
| `spaceSm` | 8dp | 表单项内部间距 |
| `spaceMd` | 16dp | 页面左右内边距 |
| `spaceLg` | 24dp | 分区间距 |
| `spaceXl` | 32dp | 大分区间距 |

### 5.2 尺寸

| 元素 | 规格 |
|---|---|
| AppBar 高度 | 56dp |
| BottomNav 高度 | 64dp + safe area |
| 主按钮高度 | 48dp |
| 图标按钮触控区 | 48dp |
| 输入框最小高度 | 48dp |
| 普通卡片圆角 | 18dp |
| 快捷操作卡片圆角 | 22dp |
| 首页 Hero 卡片圆角 | 26dp |
| 图片缩略图圆角 | 8dp |
| 页面横向内边距 | 16dp |

规则:

- 所有可点击区域在 Android 上不小于 48dp。
- 固定底部栏必须给滚动内容预留底部 inset。
- 页面不要出现横向滚动。
- 不做卡片套卡片。

## 6. 核心页面设计

### 6.1 首次建档页

目标:让用户 5 分钟内完成第一只兔子的建档。

布局:

- 顶部:简短标题和当前步骤。
- 中部:头像选择、名字、性别、生日或领养日、品种、毛色。
- 底部:固定主按钮“完成建档”。

交互:

- 头像可跳过。
- 必填项缺失时,错误提示显示在字段附近。
- 日期使用系统日期选择器。
- 性别用分段控件。
- 品种和毛色 v0.1 可以先用文本输入,后续再做预置选择。

### 6.2 记录页

目标:让用户快速看到最近记录,并随手新增。

布局:

- 顶部:当前兔子简要信息,包含头像、名字、年龄/领养时长。
- 主体:时间轴列表。
- 右下:新增日记 FAB。

日记卡片:

- 第一行:日期、标签、更多操作。
- 第二行:正文,超过 3 行折叠。
- 图片区:1 张大图、2-4 张网格、5-9 张三列网格。
- 底部:编辑、删除等低强调操作放入更多菜单。

空状态:

- 展示简短文案和“写第一条日记”按钮。
- 空状态图可用简单线性插画,不要用 emoji。

### 6.3 日记编辑页

目标:输入阻力低,不让用户像填表一样写日记。

布局:

- 顶部 AppBar:取消、标题、保存。
- 正文输入放在首屏主要区域。
- 图片选择区在正文下方。
- 日期和标签放在下方设置区。

规则:

- 正文和照片至少有一个。
- 照片最多 9 张。
- 保存时按钮进入 loading 状态,防止重复提交。
- 删除照片需要清晰的移除按钮。
- 保存成功后返回时间轴。

### 6.4 体重页

目标:让用户快速理解体重变化,而不是做复杂数据分析。

布局:

- 顶部:当前最新体重、最近记录日期。
- 中部:体重折线图。
- 下方:历史记录列表。
- 右下:新增体重 FAB。

图表规则:

- 少于 4 条记录时,优先展示最新体重和历史列表,图表可弱化。
- 折线图使用主色 `primary`,点位使用白底描边。
- 纵轴单位固定为 g。
- 不使用仅靠颜色表达异常。

### 6.5 体重编辑页

布局:

- 日期选择
- 体重输入
- 备注输入
- 保存按钮

交互:

- 体重输入使用数字键盘。
- 体重必须大于 0。
- 单位 `g` 固定展示在输入框右侧。
- 保存失败时显示字段级或页面级错误。

### 6.6 我的页

目标:承载低频管理动作,不打扰记录主流程。

布局:

- 当前兔子档案入口
- 设置入口
- v0.2 后加入备份和提醒入口

规则:

- 不把低频功能放进记录页。
- 不展示不可点击的功能入口。

## 7. 组件规范

### 7.1 按钮

| 类型 | 用途 | 规格 |
|---|---|---|
| Primary Button | 保存、完成建档 | 高 48dp,主色背景 |
| Secondary Button | 次要动作 | 边框或浅色背景 |
| Text Button | 取消、轻量操作 | 无背景,保留 48dp 触控区 |
| Icon Button | 返回、更多、删除照片 | 48dp 触控区,有语义标签 |
| FAB | 新增日记、新增体重 | 使用平台标准 FAB |

规则:

- 每个页面只放一个最强主操作。
- loading 时禁用重复点击。
- 删除动作使用危险色并二次确认。

### 7.2 输入框

规则:

- 必须有可见标签。
- 错误文案显示在字段下方。
- 占位符只做示例,不能替代标签。
- 数字输入使用数字键盘。
- 长文本输入支持多行自适应。

### 7.3 Chip 和标签

用途:

- 日记标签
- 性格标签
- 里程碑标签

规则:

- Chip 高度不低于 32dp。
- 可选中态必须通过背景、边框和文字同时体现。
- 里程碑标签用 `secondary` 或图标标识,但不使用 emoji。

### 7.4 卡片和列表

规则:

- 普通内容卡片圆角 18dp,快捷操作卡片圆角 22dp,首页 Hero 卡片圆角 26dp。
- 图片、缩略图和小型媒体容器圆角 8dp。
- 列表项高度稳定,动态内容不引发布局跳动。
- 卡片只用于单条日记、体重记录等重复项。
- 页面分区不用大卡片包裹。

## 8. 交互规则

### 8.1 导航

- 底部导航只承载一级页面。
- 返回行为必须可预测:编辑页保存后回到来源列表,取消回到来源页。
- 不用手势作为唯一操作方式。
- 页面深层动作通过 AppBar 返回或底部按钮完成。

### 8.2 动效

- 页面切换使用平台默认动效。
- 按压反馈在 80-150ms 内出现。
- 微交互时长控制在 150-300ms。
- 不做纯装饰动画。
- 支持减少动态效果。

### 8.3 反馈

- 保存、删除、图片复制等异步操作必须有 loading 或禁用态。
- 成功反馈可以用轻量 toast/snackbar。
- 错误反馈必须可定位到问题。
- 删除兔子、删除日记需要确认。

## 9. 无障碍与可用性

必须满足:

- 文字对比度达到 WCAG AA。
- 所有图标按钮有可读语义。
- 触控目标不小于 48dp。
- 表单字段有标签、提示和错误。
- 色彩不是唯一信息载体。
- 支持系统字号放大。
- 图片有业务语义时提供可访问描述;纯装饰图不进入朗读顺序。

---

## 10. Flutter 实现约束

- 使用 `MaterialApp.router` 和 go_router。
- 使用 Flutter `Form` + `GlobalKey` 做表单校验。
- 使用语义化 Theme token,不要在业务组件里散落硬编码颜色。
- 使用 `Scaffold`、`NavigationBar`、`FloatingActionButton`、`SnackBar` 等平台组件。
- UI 图标统一使用共享 `RabySketchIcon`;业务页面不直接混用系统 `Icons.*`。
- 图片列表使用稳定尺寸和懒加载策略。
- 长列表后续需要虚拟化/分页思路,v0.1 先保持列表结构可替换。
- 图表使用 fl_chart,并保留数据表/列表作为可访问替代。

## 11. UI 验收清单

- [ ] 底部导航在小屏上不拥挤
- [ ] 所有按钮和图标按钮触控区域不小于 48dp
- [ ] 记录页 1/2/4/9 张图片布局都稳定
- [ ] 日记正文长文本不会撑破卡片
- [ ] 体重图表少数据和多数据状态都可读
- [ ] 表单错误显示在字段附近
- [ ] 删除动作有确认
- [ ] 退出重进后页面状态和数据一致
- [ ] 主要页面支持系统字体放大
- [ ] 浅色主题下文字、图标、边框对比清晰
- [ ] App 内导航、操作、状态、表单和空状态图标属于同一套手绘线稿风格
- [ ] 没有 emoji 作为结构性图标
- [ ] 没有大面积单一棕橙/奶油色主题
