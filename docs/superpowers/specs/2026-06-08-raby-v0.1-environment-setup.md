## 3. Android 构建补齐项

为保证 Flutter Android debug 构建可在当前网络环境下完成,已补齐以下 SDK 组件:

| 组件 | 安装位置 | 说明 |
|---|---|---|
| NDK r28c | `C:\Users\ruihuang13\AppData\Local\Android\Sdk\ndk\28.2.13676358` | `sqlite3_flutter_libs` / `jni` 原生构建需要 |
| CMake 3.22.1 | `C:\Users\ruihuang13\AppData\Local\Android\Sdk\cmake\3.22.1` | Gradle 原生任务需要精确版本 `cmake;3.22.1` |
| Build Tools 36.0.0 | `C:\Users\ruihuang13\AppData\Local\Android\Sdk\build-tools\36.0.0` | Flutter/AGP 构建链会请求该版本 |
| Platform android-36 | `C:\Users\ruihuang13\AppData\Local\Android\Sdk\platforms\android-36` | App compileSdk 依赖 |
| Platform android-35 | `C:\Users\ruihuang13\AppData\Local\Android\Sdk\platforms\android-35` | 部分插件编译链会请求 |

CMake 3.22.1 Windows 包来源为腾讯 Android SDK 镜像:

```text
https://mirrors.cloud.tencent.com/AndroidSDK/cmake-3.22.1-windows.zip
SHA1:292778f32a7d5183e1c49c7897b870653f2d2c1b
```

当前网络下直连 Google SDK 仓库不稳定,已做两类镜像配置:

- 项目 `android/settings.gradle.kts`、`android/build.gradle.kts` 使用 Aliyun/Tencent Maven 镜像。
- Flutter SDK 内部 Gradle 插件解析文件 `D:\work\toolchains\flutter\packages\flutter_tools\gradle\settings.gradle.kts` 已加入同样的 Maven 镜像。
- 用户级 Gradle init 脚本 `C:\Users\ruihuang13\.gradle\init.d\raby-maven-mirrors.gradle` 为 buildscript 仓库注入镜像。

`android/gradle.properties` 额外固定:

```properties
kotlin.incremental=false
kotlin.compiler.execution.strategy=in-process
android.newDsl=false
android.builtInKotlin=false
```

其中 Kotlin 相关配置用于规避 Windows 跨盘缓存路径导致的 Kotlin 增量编译问题。
