## 1. 风格命名

Raby 的项目视觉主题命名为:

**Raby Warm MD3**

中文描述:

**温暖记录工具风**

它不是纯 Material You,也不是纯手绘或纯宠物可爱风。它的组成比例建议为:

| 来源 | 占比 | 用途 |
|---|---:|---|
| Material You (MD3 Mobile) | 60% | 页面结构、圆角、底部导航、表单、触控标准 |
| Flat Design Mobile | 25% | 清晰层级、低装饰、稳定布局 |
| Sketch Hand-Drawn | 10% | 兔子头像、App UI 图标、空状态、少量纸质/手绘细节 |
| Claymorphism | 5% | 主卡片和关键按钮的轻微软体积感 |

一句话标准:

**像一个认真、温暖、长期能用的兔子生活记录本,而不是一个花哨宠物社区。**

## 2. 设计原则

### 2.1 必须坚持

- 内容优先:日记、照片、体重数据始终比装饰更重要。
- 温暖但克制:用色柔和,但不大面积奶油/棕橙。
- 可长期看:避免强刺激色、复杂纹理、过度动画。
- Android 友好:优先遵循 Material 3 的控件和触控规范。
- 轻手绘:手绘感只作为品牌记忆点,不能影响信息读取。
- 图标统一:App 内导航、操作、状态、表单和空状态图标统一使用 Raby 手绘线稿风格。
- 可访问:触控目标不小于 48dp,文字对比满足 WCAG AA。

### 2.2 明确避免

- 不用 emoji 当导航、按钮或状态图标。
- 不混用 Material Icons、emoji 图标、贴纸图标和手绘图标。
- 不做大面积渐变背景。
- 不做玻璃拟态、霓虹、赛博、高饱和块状风。
- 不做厚重黑色描边和贴纸式 Neo Brutalism。
- 不把所有卡片都做成软 3D 泡泡。
- 不使用幼儿化字体或过多插画。
- 不用颜色作为唯一状态表达。

## 3. 色彩系统

### 3.1 核心色板

| Token | Hex | 用途 |
|---|---|---|
| `primary` | `#2F6F64` | 主按钮、底部导航选中态、关键进度、主图表线 |
| `onPrimary` | `#FFFFFF` | 主色背景上的文字和图标 |
| `secondary` | `#92400E` | 日记感强调、里程碑、纸质标签 |
| `onSecondary` | `#FFFFFF` | 次色背景上的文字和图标 |
| `accent` | `#6366F1` | 图表辅助线、少量信息高亮 |
| `background` | `#FAFAF7` | App 页面背景 |
| `paper` | `#FDFBF7` | 纸质卡片、回忆、里程碑区域 |
| `surface` | `#FFFFFF` | 卡片、输入框、底部导航 |
| `surfaceSoft` | `#EEF3F0` | 选中态、轻量分区背景 |
| `surfaceWarm` | `#FFF7ED` | 日记标签、温暖提示 |
| `border` | `#DDE7E2` | 常规边框、分割线 |
| `borderWarm` | `#F1E8E2` | 纸质区域边框 |
| `textPrimary` | `#17211D` | 主文字 |
| `textSecondary` | `#64748B` | 次级文字、时间、辅助说明 |
| `danger` | `#DC2626` | 删除、错误 |
| `success` | `#059669` | 成功状态 |
| `warning` | `#B45309` | 体重关注提醒 |

### 3.2 使用规则

- `primary` 只用于主操作、当前选中和核心数据,不要铺满页面。
- `secondary` 只用于日记感和里程碑,不要和 `primary` 抢主层级。
- `accent` 用量要少,优先在图表或高亮信息中使用。
- 页面背景使用 `background`,卡片使用 `surface`。
- 纸质感区域用 `paper` + `borderWarm`,不要额外加纹理噪点。
- 删除、错误必须使用 `danger`,并配明确文字。

### 3.3 颜色比例

推荐页面色彩占比:

| 类型 | 占比 |
|---|---:|
| 背景/留白 | 55-65% |
| 白色卡片/表单 | 20-30% |
| 主色青绿 | 8-12% |
| 日记棕/暖色 | 3-6% |
| 辅助紫/状态色 | 1-3% |

## 4. 字体与排版

### 4.1 字体

v0.1 使用 Flutter/Android 系统字体:

- Android 默认:Roboto / system sans
- 不额外引入 Google Fonts
- 不使用手写字体作为正文

后续如果需要品牌字体,只能用于 logo 或极少量品牌标题,不能影响表单和正文可读性。

### 4.2 字号

| Token | 规格 | 用途 |
|---|---|---|
| `displaySmall` | 26sp / 800 | 首页主卡片标题 |
| `titleLarge` | 22-24sp / 800 | 页面标题 |
| `titleMedium` | 18sp / 700 | 分区标题 |
| `titleSmall` | 16sp / 700 | 卡片标题 |
| `bodyLarge` | 16sp / 400 | 正文 |
| `bodyMedium` | 15sp / 400 | 日记正文、说明 |
| `labelLarge` | 14sp / 700 | 按钮、导航 |
| `labelMedium` | 12-13sp / 700 | 标签、时间、辅助信息 |

### 4.3 排版规则

- 正文不小于 15sp。
- 标题使用 700-850 字重,但不使用负字距。
- 中文正文行高保持 1.45-1.6。
- 长文本优先换行,不硬截断。
- 日记正文卡片默认最多展示 3 行,详情页完整展示。

## 5. 圆角与形状

### 5.1 圆角 Token

| Token | 值 | 用途 |
|---|---:|---|
| `radiusXs` | 8dp | 图片、普通小卡片 |
| `radiusSm` | 12dp | Chip、轻量输入区 |
| `radiusMd` | 16dp | 按钮、图标按钮 |
| `radiusLg` | 18dp | 普通卡片 |
| `radiusXl` | 22dp | 快速操作卡片 |
| `radiusHero` | 26dp | 首页主卡片、重点模块 |
| `radiusPill` | 999dp | 标签、分段控件 |

### 5.2 使用规则

- 卡片圆角整体偏圆,符合 MD3 亲和感。
- 表单和按钮圆角统一,避免一个页面里混用太多半径。
- 兔子头像和品牌小图标可以轻微旋转,但业务卡片不旋转。
- 不使用 0 圆角或厚重硬边框作为主风格。

## 6. 阴影与层级

### 6.1 阴影 Token

| Token | CSS 参考 | Flutter 含义 |
|---|---|---|
| `shadowSoft` | `0 10px 28px rgba(23, 33, 29, 0.08)` | 普通卡片轻阴影 |
| `shadowNav` | `0 12px 30px rgba(23, 33, 29, 0.14)` | 底部导航 |
| `shadowClay` | `0 12px 0 rgba(47, 111, 100, 0.10)` | 重点卡片的软体积感 |
| `shadowSketch` | `4px 4px 0 rgba(47, 111, 100, 0.18)` | 手绘小图标/头像 |

### 6.2 使用规则

- 大多数卡片使用轻阴影或只有边框。
- `shadowClay` 只用于首页主卡片、关键快捷操作,不能泛滥。
- `shadowSketch` 只用于品牌图标、兔子头像、空状态插画。
- 不做强玻璃反射、不做多层复杂投影。

## 7. 组件风格

### 7.1 App Scaffold

- 背景使用 `background`。
- 页面横向 padding 为 16dp。
- 页面滚动内容底部必须避开底部导航。
- v0.1 底部导航为 3 项:记录、体重、我的。
- v0.2 增加分析后为 4 项。

### 7.2 顶部区域

首页顶部包含:

- 左侧品牌/当前兔子入口
- 右侧通知或设置图标按钮

规则:

- 高度不低于 48dp。
- 图标按钮触控区 48dp。
- 品牌图标可以带轻微手绘描边和 2 度以内旋转。

### 7.3 Hero 卡片

用于首页当前兔子概览。

规格:

- 圆角 26dp
- 内边距 18dp
- 背景可用极浅青绿渐变,但不能艳
- 右侧允许兔子头像插画
- 只显示 1 个主信息标题和 1 段摘要

规则:

- 不堆多个指标。
- 不使用真实照片做 Hero 背景。
- 手绘兔子只做轻量品牌记忆点。

### 7.4 快捷操作卡片

用于“写生活日记”“记录体重”。

规格:

- 双列布局
- 高度不低于 86dp
- 主操作用 `primary` 实底
- 次操作用 `surface` + `border`
- 图标使用 Raby 手绘线稿图标

规则:

- 每个快捷卡只有一个动词。
- 主次关系明确。
- 不使用图标 emoji。
- 图标和文字必须居中对齐,图标只辅助识别,不能抢过文字。

### 7.5 内容卡片

用于日记、体重记录、提示。

规格:

- 圆角 18dp
- 内边距 14-16dp
- 边框 `border`
- 默认 `surface`
- 可使用 `shadowSoft`

规则:

- 卡片之间垂直间距 12-16dp。
- 卡片内部文字层级最多 3 层。
- 不做卡片套卡片。

### 7.6 标签 Chip

规格:

- 高度不低于 28dp
- 圆角 pill
- 字号 12-13sp
- 选中态使用 `surfaceSoft` + `primary`
- 里程碑使用 `surfaceWarm` + `secondary`

规则:

- 标签可换行。
- 标签文字不超过 6 个中文字符为佳。
- 里程碑不使用星星 emoji,后续用专用线性图标。

### 7.7 底部导航

规格:

- 高度 68dp + safe area
- 圆角 24dp
- 背景 `surface` 或 92% 白色
- 选中态 `surfaceSoft`
- 图标 21-24dp
- 文案 11-12sp

规则:

- 不超过 4 项。
- 选中态通过背景、图标色、文字色同时表达。
- 不把 FAB 嵌入底部导航中央。

## 8. 图标与插画

### 8.1 图标

Raby 的 App 内 UI 图标统一采用 **Raby Sketch Icons**:基于线性图标语义,但带轻微手绘感的自绘线稿图标。它们用于导航、按钮、表单、状态、日期、照片、标签、更多菜单、错误和空状态等日常 UI。

规则:

- 使用共享手绘线稿图标组件,当前实现为 `lib/shared/widgets/raby_sketch_icon.dart` 中的 `RabySketchIcon`。
- 统一 2.4-2.8dp 左右的线稿视觉重量,端点和转角使用圆角。
- 图标允许轻微不规则感,但轮廓必须清楚,不能影响快速识别。
- 导航、主操作、表单、状态和空状态图标必须保持同一套风格。
- 图标按钮必须有语义标签。
- 不使用 emoji 作为图标。
- 不直接混用系统 Material Icons、Lucide、贴纸或照片图标。

尺寸:

| 场景 | 建议尺寸 |
|---|---:|
| 文本按钮 / Chip 内小图标 | 18dp |
| 底部导航 / 常规图标按钮 | 21-24dp |
| 首页主操作按钮 | 24-26dp |
| 空状态 / 卡片强调图标 | 28-44dp |

颜色:

- 主操作实底上使用 `onPrimary`。
- 普通操作、导航选中态、体重和健康相关图标使用 `primary`。
- 日期、日记标签、温暖提示可使用 `secondary`。
- 错误、删除等危险动作使用 `danger`。
- 禁用态跟随组件禁用色,不能使用高对比主色。

形态:

- 线条要像“手绘线稿”,不是机械几何图标。
- 图标内部细节要少,优先保证 24dp 下可读。
- 除 `more` 这类点状图标外,默认不使用大面积实心填充。
- 兔子、体重、日记、照片、日期、趋势等核心语义需要有专用图标,不能用临时通用图标长期替代。
- 图标不单独承担状态表达,选中、危险、禁用等状态必须同时通过颜色、背景、文字或边框表达。

### 8.2 兔子插画

使用场景:

- 首页 Hero
- 首次建档空状态
- 空日记状态
- 备份成功/失败等低频反馈

风格:

- 黑色或深墨色线稿
- 白色填充
- 轻微手绘感
- 可搭配 `shadowSketch`

限制:

- 插画不进入所有卡片。
- 插画不替代真实照片。
- 不使用复杂毛发、阴影或拟真兔子。

## 9. 图片与媒体

### 9.1 日记图片

布局规则:

- 1 张:大图,16dp 圆角。
- 2-4 张:2 列网格。
- 5-9 张:3 列网格。
- 图片间距 8dp。
- 缩略图必须有稳定比例,避免加载跳动。

### 9.2 占位图

图片加载前:

- 使用 `surfaceSoft` 背景。
- 可以有极淡青绿/紫色渐变。
- 不显示破图图标。

加载失败:

- 显示明确错误文案或占位状态。
- 提供重新选择/删除路径。

---

## 10. 图表风格

### 10.1 体重折线图

规则:

- 主线使用 `primary`。
- 面积填充使用 `primary` 10-14% 透明度。
- 点位使用白底 + 主色描边。
- 坐标轴弱化,但不能完全隐藏关键刻度。
- 单位固定为 g。

### 10.2 数据不足状态

- 少于 4 条记录时,不要强行展示复杂折线图。
- 优先展示最新体重、最近日期和历史列表。
- 给用户明确的下一步:继续记录。

## 11. 动效与触感

### 11.1 动效规则

- 按压反馈在 80-150ms 内出现。
- 页面内微交互时长 150-300ms。
- 页面切换使用平台默认动画。
- 主按钮 loading 时禁用重复点击。
- 不做循环装饰动画。

### 11.2 允许的动效

- 按钮轻微透明度变化。
- 卡片轻微 elevation/scale 反馈。
- 列表新增时轻微 fade/slide。
- 图表首次出现时轻微绘制动画。

### 11.3 禁止的动效

- 背景持续动。
- 大范围弹跳。
- 文字 marquee。
- 装饰图形漂浮。
- 影响阅读的旋转和晃动。

## 12. Flutter 落地规范

### 12.1 Theme Extension

建议建立 `RabyColors` 和 `RabyRadii`:

```dart
@immutable
class RabyColors extends ThemeExtension<RabyColors> {
  const RabyColors({
    required this.paper,
    required this.surfaceSoft,
    required this.surfaceWarm,
    required this.borderWarm,
  });

  final Color paper;
  final Color surfaceSoft;
  final Color surfaceWarm;
  final Color borderWarm;
}
```

### 12.2 Material 3

Flutter 主题必须开启:

```dart
ThemeData(
  useMaterial3: true,
  colorScheme: ColorScheme.fromSeed(
    seedColor: const Color(0xFF2F6F64),
  ),
)
```

再用显式 token 覆盖关键颜色,避免 seed 自动色偏太大。

### 12.3 组件实现要求

- 使用 `NavigationBar` 做底部导航。
- 使用 `FilledButton` / `OutlinedButton` / `TextButton` 做按钮。
- 使用 `Card` 或自定义 `Container` 统一卡片样式。
- 表单使用 `TextFormField` + `Form`。
- 日记列表使用可替换为分页/懒加载的列表结构。
- 图片组件必须固定尺寸或 aspect ratio。

低频管理页:

- 保持安静。
- 不堆营销信息。
- 入口列表清楚,每个入口有图标和说明。

## 14. 风格验收清单

- [ ] 页面第一眼是温暖记录工具,不是宠物社区或医疗软件。
- [ ] 主色青绿使用克制,没有大面积铺满。
- [ ] 日记棕只用于里程碑、标签、纸质强调。
- [ ] 所有卡片圆角和阴影一致。
- [ ] 兔子插画是轻量线稿,没有过度可爱化。
- [ ] App 内图标统一为 Raby 手绘线稿风格,没有混用系统图标或贴纸图标。
- [ ] 没有 emoji 作为结构性图标。
- [ ] 底部导航选中态清晰。
- [ ] 所有可点击区域不小于 48dp。
- [ ] 长文本不会挤压布局。
- [ ] 图片加载前后不造成明显跳动。
- [ ] 图表不只靠颜色传达含义。
- [ ] 表单错误显示在字段附近。
- [ ] 删除、危险动作有明确确认。
- [ ] 系统字号放大后仍可读。

## 15. 快速判断

如果一个设计方案出现下面特征,说明偏离了 Raby Warm MD3:

| 现象 | 问题 |
|---|---|
| 满屏奶油黄、棕色、橙色 | 过度日记/咖啡馆化 |
| 大量贴纸、手绘箭头、胶带 | 过度手账化 |
| 强烈渐变和玻璃模糊 | 偏 SaaS/科技炫技 |
| 黑粗边框和硬阴影 | 偏 Neo Brutalism |
| 大面积兔子插画 | 内容被装饰抢走 |
| 表单像后台管理系统 | 温暖感不足 |
| 图表色彩很多 | 数据噪音过高 |

Raby 的正确方向是:

**清楚、柔和、轻手绘、有一点软体积,但永远先服务记录。**
